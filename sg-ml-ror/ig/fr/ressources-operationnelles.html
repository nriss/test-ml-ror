Cette section est transversale : elle décrit les moyens mis en œuvre pour réaliser les prestations de soins.

La section Ressources Opérationnelles décrit les moyens matériels et capacitaires qui peuvent être mis en œuvre pour réaliser une offre opérationnelle. Elle comprend les équipements spécifiques, les capacités d'accueil et le lieu de réalisation de l'offre.

Donnée pivot

L'identifiant du lieu de réalisation de l'offre est la donnée pivot de cette section. Lorsqu'il est généré par une instance régionale, il est de la forme code INSEE régional / code interne ROR.

RORLieuRealisationOffre

Le Lieu de Réalisation de l'Offre indique le lieu précis où se déroule la prestation. Il hérite de la classe MOS Lieu (adresse, coordonnées géographiques, etc.) et ajoute les attributs spécifiques au contexte ROR.

ror-lieu-realisation-offre : LieuRealisationOffre
Spécialisation ROR du LieuRealisationOffre du MOS. Espace disposant d'un ensemble de ressources pour réaliser une offre. Lorsque les ressources sont mobiles, le lieu correspond au lieu administratif où est organisée la prise en charge.
commentaire 0..1 string Commentaire permettant de donner des informations complémentaires sur le lieu.
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données du lieu de réalisation dans le ROR.

ROREquipementSpecifique

L'Équipement Spécifique décrit les équipements matériels et installations spécialisés disponibles dans un établissement, susceptibles d'influencer l'orientation d'un patient.

ror-equipement-specifique : EquipementSpecifique
Spécialisation ROR de l'EquipementSpecifique du MOS. Ressource matérielle discriminante pour la réalisation d'une prestation, propre à la structure ou mise à disposition via convention (sur site).
typeEquipement 1.. 1 Coding Type d'équipement (obligatoire dans le ROR)
Binding: JDVROREquipement (0.1.0) ( required )
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données de l'équipement dans le ROR.

RORCapacitePriseCharge

La Capacité de Prise en Charge décrit les capacités d'accueil en termes de places autorisées ou installées pour une offre donnée.

ror-capacite-prise-charge : CapacitePriseCharge
Spécialisation ROR de la CapacitePriseCharge du MOS. Indique l'affectation des lits d'une zone d'hébergement, notamment dans le cadre d'une gestion de crise.
affectationTemporaire 1.. 1 Coding Affectation temporaire des lits (obligatoire dans le ROR)
Binding: JDVRORAffectationTemporaire (0.1.0) ( required )
commentaire 0..1 string Commentaire permettant de donner des indications complémentaires sur les informations capacitaires.
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données de la capacité dans le ROR.

RORCapaciteAccueilOperationnelle

La Capacité d'Accueil Opérationnelle représente les disponibilités de prise en charge en temps réel ou à court terme. Elle permet aux outils d'orientation d'accéder aux disponibilités d'un établissement.

Règles de gestion :

  • RG_EXP_047 : Lorsque le statutCapacite est à « disponible » avec une valeur à 0, cela indique que l'offre est actuellement saturée mais reste ouverte.
ror-capacite-accueil-operationnelle : Base
Précise la CapacitePriseCharge par une série d'enregistrements indiquant la quantité de lits (ou places) pour un statut et une temporalité donnée. Le cumul des enregistrements donne une vision complète de la capacité d'un lieu.
natureCapacite 1..1 Coding Indique si la capacité est exprimée en lits ou en places.
Binding: JDVRORNatureCapacite (0.1.0) ( required )
statutCapacite 1..1 Coding Caractérise la capacité d'accueil et précise le statut des lits décrits.
Binding: JDVRORStatutCapacite (0.1.0) ( required )
temporaliteCapacite 1..1 Coding Indique le moment où cette capacité sera effective (situation immédiate ou prospective).
Binding: JDVRORTemporalite (0.1.0) ( required )
nombreCapacite 1..1 integer Quantité de lits ou places de la capacité exprimée.
typeSourceCapacite 1..1 Coding Indique quel type de source est à l'origine de l'information de capacité.
Binding: JDVRORTypeSource (0.1.0) ( required )
dateMAJCapacite 1..1 dateTime Date à laquelle la capacité d'accueil a été mise à jour dans la source.
genreCapaciteDispo 0..1 Coding Genre des patients pouvant être installés dans des lits disponibles.
Binding: JDVRORGenreCapacite (0.1.0) ( required )
typeFermetureCapacite 0..1 Coding Statut des lits fermés permettant d'identifier lesquels peuvent être réactivés.
Binding: JDVRORTypeFermetureCapacite (0.1.0) ( required )
typeLitSupplementaire 0..1 Coding Statut des lits supplémentaires identifiant lesquels sont déjà mobilisés.
Binding: JDVRORTypeLitSupplementaire (0.1.0) ( required )
typeCrise 0..1 Coding Type de crise permettant de mobiliser le nombre de lits supplémentaires décrits.
Binding: JDVRORTypeCrise (0.1.0) ( required )
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données de capacité dans le ROR.

RORCapaciteHabitation

La Capacité Habitation décrit le nombre de places ou de logements disponibles selon le type d'hébergement.

ror-capacite-habitation : CapaciteHabitation
Spécialisation ROR de la CapaciteHabitation du MOS. Décrit un type d'habitation adapté à la réalisation d'une offre et le nombre d'habitations associées.
typeHabitation 1.. 1 Coding Type d'habitation (obligatoire dans le ROR)
Binding: JDVRORTypeHabitation (0.1.0) ( required )
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données de la capacité d'habitation dans le ROR.

RORLimiteCaracteristiqueEquipement

La Limite de Caractéristique d'Équipement précise les limites que peut supporter l'équipement le plus tolérant parmi les équipements spécifiques de même type dans une organisation. Par exemple, si une organisation possède trois tables de bloc opératoire supportant respectivement 180 kg, 190 kg et 200 kg, seule la valeur de 200 kg est retenue.

ror-limite-caracteristique-equipement : Base
Précise les limites que peut supporter l'équipement le plus tolérant parmi les équipements spécifiques de même type dans cette organisation. Par exemple, si une organisation possède 3 tables de bloc opératoire pouvant accueillir des patients d'un poids maximum de 180 kg, 190 kg et 200 kg, seule la valeur limite de 200 kg sera identifiée dans cette classe.
typeCaracteristique 1..1 code Type de caractéristique de l'équipement
Binding: https://mos.esante.gouv.fr/NOS/TRE_R340-TypeCaracteristiqueEquipement/FHIR/TRE-R340-TypeCaracteristiqueEquipement|20240329120000 ( required )
valeurLimite 1..1 Quantity Valeur limite de la caractéristique
