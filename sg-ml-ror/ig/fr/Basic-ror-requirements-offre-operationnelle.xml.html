</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-007</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-activite-operationnelle.html">ROR - Activité Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_007</strong> : La <code>familleActiviteOperationnelle</code> DOIT être renseignée pour toute instance de la classe Activité Opérationnelle liée à une offre dont le <code>champActivite</code> est « médico-social ».</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-008</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-patientele.html">ROR - Patientèle</a></li></ul></blockquote><div><p><strong>RG_EXP_008</strong> : Si le <code>champActivite</code> de l'offre est « médico-social », le public pris en charge (<code>publicPrisEnCharge</code>) DOIT être renseigné dans l'instance de la classe Patientèle.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-009</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-patientele.html">ROR - Patientèle</a></li></ul></blockquote><div><p><strong>RG_EXP_009</strong> : Chaque instance de la zone d'intervention DOIT être composée par des instances de <code>DivisionTerritoriale</code> de type Commune.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-010</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_010</strong> : Une offre DOIT être renseignée avec un et un seul <code>champActivite</code>, qui est invariable.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-011</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_011</strong> : Une offre opérationnelle DOIT être renseignée avec un seul <code>modePriseEnCharge</code>, qui est invariable. Le changement de mode de prise en charge implique la création d'une nouvelle offre.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-014</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_014</strong> : Les horaires transmis DOIVENT inclure au minimum une période (date début, date fin) ou un jour de semaine.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-015</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_015</strong> : Si les horaires sont transmis sans période, ils s'appliquent aux jours de la semaine à partir du jour de la semaine de lecture de l'information dans le ROR (semaine glissante).</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-016</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_016</strong> : Les horaires de jours de semaine RESTENT les horaires de référence semaine après semaine TANT QU'aucun nouvel horaire n'est récupéré d'un ROR.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-017</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_017</strong> : Les horaires portant sur une période (date début, date fin) DOIVENT être transmis si la période est en cours ou à venir lors de la lecture de l'information dans le ROR.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-018</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_018</strong> : Les horaires portant sur une période échue (date de fin antérieure à la date du jour) NE DOIVENT PAS être transmis.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-028</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_028</strong> : Une offre opérationnelle ne peut pas avoir plusieurs types d'offre relevant de champs d'activité différents.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-035</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_035</strong> : Lors de la diffusion d'une unité élémentaire, l'indicateur <code>uniteSensible</code> a la valeur par défaut <code>false</code>.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-039</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_039</strong> : Dans une unité sensible, le niveau de confidentialité des contacts doit être positionné à la valeur « très restreint ».</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-042</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_042</strong> : Pour un transfert d'une offre d'un établissement vers un autre, l'identifiant de l'offre transférée NE DOIT PAS changer.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-043</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_043</strong> : La date de fermeture est obligatoire lorsqu'un type de fermeture est renseigné, sinon elle n'est jamais renseignée.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-044</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_044</strong> : Lorsqu'un niveau de structure ou d'organisation est fermé, tous les niveaux inférieurs ainsi que toutes les offres opérationnelles qui lui sont rattachées sont également fermés avec le même type, la même date de fermeture et, le cas échéant, la même date de réouverture prévisionnelle.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-045</li><li>requirement: </li><li>conformance: MAY</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_045</strong> : La date de réouverture peut être renseignée lorsque le type de fermeture est « Fermeture temporaire », sinon elle n'est jamais renseignée.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-046</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_046</strong> : La date de réouverture est remise à nulle lorsque l'offre est ouverte de nouveau.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-047-offre</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-offre-operationnelle.html">ROR - Offre Opérationnelle</a></li></ul></blockquote><div><p><strong>RG_EXP_047</strong> : Lorsqu'une offre opérationnelle est fermée, les ressources de type <code>statutCapacite</code> « disponible » doivent être à 0.</p>
</div><blockquote><p><b>R5: Actual statement as markdown (new)</b></p><ul><li>key: RG-EXP-048</li><li>requirement: </li><li>conformance: SHALL</li><li>satisfiedBy: <a href="StructureDefinition-ror-patientele.html">ROR - Patientèle</a></li></ul></blockquote><div><p><strong>RG_EXP_048</strong> : L'unité de mesure des attributs <code>ageMin</code> et <code>ageMax</code> accepte uniquement les valeurs : <code>a</code> (année), <code>mo</code> (mois), <code>wk</code> (semaine), <code>d</code> (jour).</p>
