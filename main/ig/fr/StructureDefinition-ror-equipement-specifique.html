Modèle logique: ROR - Équipement Spécifique

URL officielle: https://interop.esante.gouv.fr/ig/fhir/test-ror/StructureDefinition/ror-equipement-specifique Version: 0.1.0
Draft à partir de 2026-06-10 Nom computable: ROREquipementSpecifique

Spécialisation ROR de l'EquipementSpecifique du MOS. Ressource matérielle discriminante pour la réalisation d'une prestation, propre à la structure ou mise à disposition via convention (sur site).

Utilisations:

  • Ce Modèle logique n'est utilisé par aucun autre profil dans ce guide d'implémentation

Vous pouvez également vérifier les usages dans le FHIR IG Statistics

typeEquipement 1..1 Coding Type d'équipement (obligatoire dans le ROR)
Binding: JDVROREquipement (0.1.0) (required)
nbEquipementEnService 0..1 integer Nombre d'équipement du même type en état de fonctionnement.
LimiteCaracteristiqueEquipement 0..* Base Précise les limites que peut supporter l'équipement le plus tolérant parmi les équipements spécifiques de même type dans cette organisation.Par exemple, si une organisation possède 3 tables de bloc opératoire pouvant accueillir des patients d'un poids maximum de 180kg, 190kg et 200Kg, seule la valeur limite de 200Kg sera identifiée dans cette classe.
typeCaracteristique 0..1 Coding Indique la caractéristique de l'équipement pour laquelle une valeur limite est précisée.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R340-TypeCaracteristiqueEquipement/FHIR/TRE-R340-TypeCaracteristiqueEquipement?vs (preferred)
valeurLimite 0..1 Quantity Correspond à la valeur extrême associée à une caractéristique de l'équipement.
LieuRealisationOffre 1..1 LieuRealisationOffre Lien vers la classe LieuRealisationOffre
Agenda 0..* Agenda Lien vers la classe Agenda
RendezVous 0..* RendezVous Lien vers la classe RendezVous
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données de l'équipement dans le ROR.
