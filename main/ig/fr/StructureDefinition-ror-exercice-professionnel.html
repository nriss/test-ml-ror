Modèle logique: ROR - Exercice Professionnel

URL officielle: https://interop.esante.gouv.fr/ig/fhir/test-ror/StructureDefinition/ror-exercice-professionnel Version: 0.1.0
Draft à partir de 2026-06-09 Nom computable: RORExerciceProfessionnel

Spécialisation ROR de l'ExerciceProfessionnel du MOS. Informations décrivant la profession exercée et l'identité d'exercice d'un professionnel.

Utilisations:

  • Ce Modèle logique n'est utilisé par aucun autre profil dans ce guide d'implémentation

Vous pouvez également vérifier les usages dans le FHIR IG Statistics

profession 1..1 Coding Profession exercée (obligatoire dans le ROR)
Binding: JDVRORProfessionSante (0.1.0) (required)
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données d'exercice professionnel dans le ROR.

civiliteExercice 0..1 Coding Civilité d’exercice du professionnel.
Binding: JDVRORCiviliteExercice (0.1.0) (required)
nomExercice 1..1 string Nom sous lequel exerce le professionnel (obligatoire dans le ROR)
prenomExercice 0..1 string Prénom sous lequel exerce le professionnel.
profession 1..1 Coding Profession exercée (obligatoire dans le ROR)
Binding: JDVRORProfessionSante (0.1.0) (required)
typeProfession 0..1 Coding Indique le type de profession de la personne.Exemples :** Professionnel de santé** Usager de titre** Professionnel secteur social** etc.
Binding: TypeprofessionVs (0.1.0-ballot) (preferred)
categorieProfessionnelle 0..1 Coding Indique si le professionnel exerce sa profession en tant que :** M: Agent public** C: Civil** F: Fonctionnaire ** E: EtudiantL'autorité d'enregistrement du professionnel dépend de la catégorie de sa profession.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R09-CategorieProfessionnelle/FHIR/TRE-R09-CategorieProfessionnelle?vs (preferred)
dateEffetExercice 0..1 date Date à partir de laquelle le professionnel exerce cette profession.
dateFinEffetExercice 0..1 date Date à partir de laquelle le professionnel n’exerce plus cette profession.
departementPremierReferencementAE 0..1 Coding Code du département du premier référencement, pour un référencement de nature « inscription à l’ordre ».
Binding: https://mos.esante.gouv.fr/NOS/TRE_G09-DepartementOM/FHIR/TRE-G09-DepartementOM?vs (preferred)
boiteLettresMSS 0..* BoiteLettreMSS Boîte(s) aux lettres du service de messagerie sécurisée de santé (MSS) rattachée(s) à l'exercice professionnel du professionnel.
InscriptionOrdre 0..* Base Eléments permettant de retrouver les informations d'inscription à un ordre par rapport à la profession de la personne physique sur une période et un département donnéSynonymes RPPS : Inscription à ordre
ordre 0..1 Coding Instance de régulation d'une profession de santé réglementée.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R82-Ordre/FHIR/TRE-R82-Ordre?vs (preferred)
dateDebutInscription 0..1 date Date d’inscription de la personne au tableau de l’ordre.
dateRadiation 0..1 date Date à partir de laquelle la personne n’est plus autorisée à exercer la profession pour laquelle elle a été inscrite.
statutInscription 0..1 Coding Information indiquant l’avancement du traitement du dossier d'inscription de la personne.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R33-StatutInscription/FHIR/TRE-R33-StatutInscription?vs (preferred)
conseilDepartemental 0..1 Coding Département du Conseil départemental.
Binding: https://mos.esante.gouv.fr/NOS/TRE_G09-DepartementOM/FHIR/TRE-G09-DepartementOM?vs (preferred)
AttributionParticuliere 0..1 Base Activités ponctuelles du professionnel de type expertise.
attributionParticuliere 0..1 Coding Attribution particulière.Par exemple : **Agrément : Autorisation d’un médecin généraliste ou spécialiste, formalisée par un agrément, à figurer sur une liste départementale établie par le Préfet sur proposition du directeur départemental de l'action sanitaire et sociale et après avis du conseil départemental de l'ordre des médecins et des syndicats départementaux des médecins.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R03-AttributionParticuliere/FHIR/TRE-R03-AttributionParticuliere?vs (preferred)
dateDebutReconnaissance 0..1 date Date à partir de laquelle l'attribution particulière prend effet.
dateAbandon 0..1 date Date à laquelle le professionnel ne veut plus faire apparaître cette attribution.
NiveauFormation 0..* Base Le niveau de formation décrit le niveau de formation de l’étudiant, comme demandé par le décret n°2010-701 et complété par le décret n°2011-462.
diplome 0..1 Diplome Diplôme acquis pour les cycles terminés (cas des diplômes de deuxième cycle pour étudiants qui font un troisième cycle) ou diplôme préparé pour les cycles en cours.
natureCycleFormation 0..1 Coding Nature du cycle de formation:- Deuxième cycle,- Troisième cycle court,- Troisième cycle long/Internat.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R223-NatCycleForm/FHIR/TRE-R223-NatCycleForm?vs (preferred)
niveauFormationAcquis 0..1 Coding Niveau de validité dans le cycle de formation.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R224-NiveauFormAcquis/FHIR/TRE-R224-NiveauFormAcquis?vs (preferred)
anneeUniversitaire 0..1 Coding Année universitaire, exemple : 2015-2016.
Binding: https://mos.esante.gouv.fr/NOS/TRE_R225-AnneeUniversitaire/FHIR/TRE-R225-AnneeUniversitaire?vs (preferred)
dateFin 0..1 date Date à laquelle le niveau de formation n’est plus actif (non visible hormis dans les données historisées). Cette date est renseignée par l’ordre à la clôture de l’exercice professionnel.
SituationExercice 0..* SituationExercice Lien vers la classe SituationExercice
Certificat 0..* Certificat Lien vers la classe Certificat
SavoirFaire 0..* SavoirFaire Lien vers la classe SavoirFaire
CarteProfessionnel 0..* CarteProfessionnel Lien vers la classe CarteProfessionnel
Professionnel 1..1 Professionnel Lien vers la classe Professionnel
SituationOperationnelle 0..* SituationOperationnelle Lien vers la classe SituationOperationnelle
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données d'exercice professionnel dans le ROR.
