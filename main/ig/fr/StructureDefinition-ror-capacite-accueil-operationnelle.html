Modèle logique: ROR - Capacité d'Accueil Opérationnelle

URL officielle: https://interop.esante.gouv.fr/ig/fhir/test-ror/StructureDefinition/ror-capacite-accueil-operationnelle Version: 0.1.0
Draft à partir de 2026-06-10 Nom computable: RORCapaciteAccueilOperationnelle

Précise la CapacitePriseCharge par une série d'enregistrements indiquant la quantité de lits (ou places) pour un statut et une temporalité donnée. Le cumul des enregistrements donne une vision complète de la capacité d'un lieu.

Utilisations:

  • Ce Modèle logique n'est utilisé par aucun autre profil dans ce guide d'implémentation

Vous pouvez également vérifier les usages dans le FHIR IG Statistics

ror-capacite-accueil-operationnelle 0..* Base(4.0.1) Capacité d'accueil opérationnelle (lits/places) dans le contexte ROR
Les instances de ce modèle logique peuvent être la cible d'une référence
natureCapacite 1..1 Coding Indique si la capacité est exprimée en lits ou en places.
Binding: JDVRORNatureCapacite (0.1.0) (required)
statutCapacite 1..1 Coding Caractérise la capacité d'accueil et précise le statut des lits décrits.
Binding: JDVRORStatutCapacite (0.1.0) (required)
temporaliteCapacite 1..1 Coding Indique le moment où cette capacité sera effective (situation immédiate ou prospective).
Binding: JDVRORTemporalite (0.1.0) (required)
nombreCapacite 1..1 integer Quantité de lits ou places de la capacité exprimée.
typeSourceCapacite 1..1 Coding Indique quel type de source est à l'origine de l'information de capacité.
Binding: JDVRORTypeSource (0.1.0) (required)
dateMAJCapacite 1..1 dateTime Date à laquelle la capacité d'accueil a été mise à jour dans la source.
genreCapaciteDispo 0..1 Coding Genre des patients pouvant être installés dans des lits disponibles.
Binding: JDVRORGenreCapacite (0.1.0) (required)
typeFermetureCapacite 0..1 Coding Statut des lits fermés permettant d'identifier lesquels peuvent être réactivés.
Binding: JDVRORTypeFermetureCapacite (0.1.0) (required)
typeLitSupplementaire 0..1 Coding Statut des lits supplémentaires identifiant lesquels sont déjà mobilisés.
Binding: JDVRORTypeLitSupplementaire (0.1.0) (required)
typeCrise 0..1 Coding Type de crise permettant de mobiliser le nombre de lits supplémentaires décrits.
Binding: JDVRORTypeCrise (0.1.0) (required)
metadonnee 1..1 ror-metadonnee Informations relatives à la gestion des données de capacité dans le ROR.
